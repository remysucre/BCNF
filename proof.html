Key-centric BCNF decomposition

Motivation

The textbook presentation of BCNF decomposition is FD-centric: look for a nontrivial FD X → Y with a non-superkey LHS, split on it, recurse. This forces the student to enumerate FDs (including derived ones from F+), it lets extraneous attributes sneak into the LHS and inflate the split (producing redundancy), and for general F the BCNF check in principle requires iterating over every possible LHS.

This note argues for a cleaner framing: key-centric.

Every BCNF violation is really a missing key. It is a set of attributes X that (i) determines something non-trivially, but (ii) is not already a superkey. The decomposition step is not about the FD; it is about X. Splitting R into R1 = X+ and R2 = (𝒰 \ X+) ∪ X promotes X to a key of R1 — that’s the whole point of the decomposition. BCNF just means “there is nothing left to promote.”

So the algorithm reads:

Find a set of attributes that could be made a key. Make it a key by decomposition. Repeat.

The two cases below — singleton F and general F — are the same algorithm, instantiated at the right granularity.

Setup

  • R is a relation with attribute set 𝒰.
  • F is a set of FDs.
  • X+ is the attribute closure of X ⊆ 𝒰 under F.
  • For S ⊆ 𝒰, the closure of X ⊆ S inside S is X+ ∩ S (the standard identity XπS(F+)+ = X+ ∩ S).
  • X is a superkey of S iff X+ ⊇ S.
  • An FD X → Y on S is nontrivial iff Y ⊈ X, and violates BCNF on S iff it is nontrivial and X is not a superkey of S.

Key candidates

Definition. A set X ⊆ S is a key candidate of S iff X ⊊ X+ ∩ S ⊊ S. Equivalently, X determines something in S beyond itself (so there is a nontrivial FD X → (X+ ∩ S) \ X inside S) yet X is not already a superkey of S.

BCNF as the absence of key candidates. S is in BCNF ⇔ S has no key candidate.

Proof. (⇒) If X is a key candidate, then X → (X+ ∩ S) \ X is a nontrivial FD on S with non-superkey LHS, violating BCNF. (⇐) Any nontrivial X → Y violating BCNF on S has Y ⊈ X, so X+ ∩ S ⊋ X, and X not a superkey gives X+ ∩ S ⊊ S. So X is a key candidate. ◼

The decomposition step. Given a key candidate X of S, split: S1 = X+ ∩ S,   S2 = (S \ X+) ∪ X. By construction X is a superkey of S1 (it determines all of S1), so X has been “promoted to a key” in S1. Both halves contain X and share only X: S1 ∩ S2 = X.

Everything else is a question of which key candidate to pick.

Singleton case

Assume F consists of FDs A → B with A, B ∈ 𝒰 and A ≠ B, and let GF be the directed graph with an edge A → B for each FD. Write reach(A) for the set of nodes reachable from A in GF (including A).

Lemma 1 (Closure is reachability). For singleton F, X+ = ⋃A ∈ Xreach(A).

Proof. Compute X+ stepwise: X0 = X, and Xi + 1 = Xi ∪ {C : ∃A ∈ Xi, A → C ∈ F}. Because every FD has a singleton LHS, the firing condition is “some single attribute already in Xi”.

(⊇) For A ∈ X and B ∈ reach(A), follow the path A = A0 → A1 → ⋯ → Ak = B: each edge Ai → Ai + 1 puts Ai + 1 into Xi + 1, so B ∈ X+.

(⊆) Induct on the smallest i with B ∈ Xi: either B ∈ X (empty path) or B is added via A → B ∈ F with A ∈ Xi − 1, and the inductive hypothesis extends a path from X to A into one to B. ◼

Single-attribute key candidates suffice

For singleton F, a key candidate can always be taken to be a single attribute. Concretely, S has a key candidate iff it has a single-attribute one:

Theorem. For singleton F and any S ⊆ 𝒰, S is in BCNF ⇔ ∀A ∈ S, reach(A) ∩ S ∈ {{A}, S}. I.e., each attribute either reaches nothing new inside S, or is a superkey of S — never in between.

Proof. Going from key candidates to single attributes: if X ⊆ S is a key candidate, pick any B ∈ (X+ ∩ S) \ X. By Lemma 1, B is reachable from some A0 ∈ X; since B ≠ A0 and B ∈ S, we have reach(A0) ∩ S ∋ B ≠ A0, so it is not {A0}. If it equaled S, then A0 ∈ X would make X a superkey, contradicting the key-candidate property. So A0 is a single-attribute key candidate of S.

Going the other way: a single-attribute key candidate is a key candidate. Applying the BCNF criterion above completes the equivalence. ◼

So for singleton F, the algorithm specializes to:

  1. For each A ∈ S that is an LHS of some FD in F (others have reach(A) = {A} and pass trivially), check whether reach(A) ∩ S = S.
  2. If every such A is a superkey of S, S is in BCNF — done.
  3. Otherwise pick any violating A and split on {A}: S1 = reach(A) ∩ S,   S2 = (S \ reach(A)) ∪ {A}. A is now a key of S1. Recurse.

The whole procedure reads the graph GF once. No powerset, no acyclicity required, no projection of FDs between steps.

Uniqueness when GF is a forest

BCNF decomposition is famously non-unique: different choices of violating FD can yield different final sets of sub-relations. The smallest example is a diamond: A → B, A → C, B → D, C → D. Splitting on B yields {B, D}, {A, B, C}; splitting on C yields {C, D}, {A, B, C} — genuinely different.

Acyclicity alone is not enough (the diamond is acyclic). What fails is a stronger tree-like property: D has two parents, so sibling subtrees are not disjoint. Requiring every node to have in-degree at most one — i.e., GF is a forest — rules exactly this out.

Call v internal if it has an outgoing edge in GF (i.e., v is the LHS of some FD), and define Tv = {v} ∪ children(v). Let K be the set of attributes with no incoming edge (roots of each tree, plus isolated nodes) — the unique candidate key of R.

Theorem (Uniqueness). If GF is a forest, the decomposition is uniquely determined and equals 𝒟 = {Tv : v internal} ∪ {K}, with K dropped when |K| = 1 (then K ⊆ Tv for the unique root v).

Uniqueness comes down to a one-step commutativity: if A and B are both violating in S, processing them in either order yields the same three pieces.

Nested (B ∈ reach(A), so A is an ancestor of B). Forest acyclicity gives A ∉ reach(B); forest in-degree  ≤ 1 gives reach(B) ⊆ reach(A). Processing A first then B produces reach(B) ∩ S, (reach(A) ∩ S) \ reach(B) ∪ {B}, (S \ reach(A)) ∪ {A}. Processing B first then A produces the same three sets by direct calculation (using reach(B) ⊆ reach(A) and B ∈ reach(A)).

Disjoint (A, B in different subtrees). This is where the forest hypothesis earns its keep: sibling subtrees being disjoint is exactly the in-degree- ≤ 1 property. Processing A puts B into the residue S2, where reach(B) ∩ S2 = reach(B) ∩ S is unchanged and B is still violating. Both orders yield reach(A) ∩ S, reach(B) ∩ S, (S \ (reach(A) ∪ reach(B))) ∪ {A, B}.

From the one-step commutativity, any two orderings can be bridged by finitely many adjacent swaps; the final multiset of sub-relations is an invariant of the algorithm. The characterization as {Tv} ∪ {K} follows by induction on the number of internal nodes.

Why the diamond breaks this. In the diamond, B and C are siblings with a shared descendant D: reach(B) ∩ reach(C) = {D} instead of empty. Processing B first puts D into {B, D} and removes it from the residue; processing C first puts D into {C, D}. D goes with whichever parent is chosen first — exactly the ambiguity.

General case

For arbitrary F, key candidates can be multi-attribute. The algorithm iterates over them in size order and picks the first one that works:

decompose(S):
  for k = 0, 1, 2, …, |S| − 1:
    for each X ⊆ S with |X| = k:
      if X ⊊ X⁺ ∩ S ⊊ S:                    # X is a key candidate of S
        return decompose(X⁺ ∩ S)
             ∪ decompose((S ∖ X⁺) ∪ X)
  return {S}

Call the first such X encountered the minimal key candidate of S. Its minimality has a useful form: every proper subset Y ⊊ X has Y+ ∩ S = Y. (A proper subset can’t be a superkey — else X would be too — and by the “smallest first” rule it isn’t a key candidate, so its closure inside S is itself.)

For singleton F, the minimal key candidate always has |X| = 1, recovering the single-attribute algorithm above. The powerset loop is the natural generalization when F has multi-attribute LHSs.

Non-redundancy

Definition. A decomposition 𝒟 = {S1, …, Sk} is non-redundant if no member is contained in another: for all i ≠ j, Si ⊈ Sj. The offending case is the textbook example F = {Z → W} on 𝒰 = {X, Y, Z, W}, where picking the augmented violating FD {Y, Z} → W (with extraneous Y) produces {{X, Y, Z}, {Y, Z}, {Z, W}} and {Y, Z} ⊊ {X, Y, Z}.

Theorem (Non-redundancy). For any F, the powerset algorithm produces a non-redundant decomposition.

Proof. Fix a split step: current sub-relation S, minimal key candidate X, halves S1 = X+ ∩ S and S2 = (S \ X+) ∪ X. Two elementary facts:

  • S1 ∩ S2 = X (direct calculation).
  • |S1| ≥ |X|+1 and |S2| ≥ |X|+1, since X ⊊ X+ ∩ S ⊊ S.

Key lemma. Every key candidate Y used inside the recursion on S1 or S2 satisfies Y ⊈ X.

Proof of lemma. Let T be a descendant piece (T ⊆ S1 or T ⊆ S2), and suppose Y ⊆ X.

  • If Y ⊊ X: by (⋆), Y+ ∩ S = Y. Since T ⊆ S, Y+ ∩ T ⊆ Y+ ∩ S = Y, so Y+ ∩ T = Y — Y is trivially closed in T, not a key candidate.
  • If Y = X and T ⊆ S1 ⊆ X+: then X+ ∩ T = T, so X is a superkey of T — not a key candidate.
  • If Y = X and T ⊆ S2: then X+ ∩ T ⊆ X+ ∩ S2 = X; combined with X ∩ T ⊆ X+ ∩ T, this gives X+ ∩ T = X ∩ T, which is trivial (self-closed) — not a key candidate.

So no Y ⊆ X is a key candidate in any T; every pivot used inside the recursion has some attribute outside X. ◼

From the lemma to non-redundancy. Both halves of any split contain the pivot. The lemma says each recursion-pivot contributes at least one attribute outside X to both halves, so every piece ever produced on the S1-side contains an attribute of X+ \ X, and every piece on the S2-side contains an attribute of S \ X+. In particular, any final sub-relation L1 descending from S1 has L1 ⊈ X, and any L2 from S2 has L2 ⊈ X.

Let L1, L2 be two final sub-relations whose recursion paths diverge at a split with pivot X, L1 on the S1-side and L2 on the S2-side. Then L1 ∩ L2 ⊆ S1 ∩ S2 = X, so L1 ⊆ L2 would give L1 ⊆ X, contradicting L1 ⊈ X. Symmetrically L2 ⊈ L1. Every two final sub-relations are incomparable. ◼

Where the textbook algorithm goes wrong. The textbook splits on a violating FD α → β, and α may carry extraneous attributes — attributes whose removal still leaves a valid (and still violating) FD, like Y in {Y, Z} → W when Z → W already holds. The inflated α propagates into R1, and a later split can shed the extras into a sub-relation that is then contained in R2. The key-centric version picks the smallest key candidate by construction, so (⋆) formally rules out extraneous attributes.

Summary

The FD-centric reading of BCNF decomposition obscures the actual object of interest: sets of attributes that are not yet keys. Reading the algorithm key-centrically — find an X you can promote to a key, promote it, repeat — collapses the BCNF check, the decomposition step, and the correctness arguments into a single pattern:
